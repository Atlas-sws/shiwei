# 拾味 ShiWei — 设计文档

> 一款精美的个人菜谱记录 App。
> 本文档记录设计访谈中逐条敲定的决策，是后续开发的唯一依据。

## 项目定位

把各种菜谱（自创的、网上看来的、家传的）收集、整理、记录下来，
**只在手机上使用**：随时录入，做菜时在厨房查看。

## 已确定的决策

- **0 · 项目名**：**拾味**（ShiWei），谐音"食味"，目录名 `shiwei`
- **1 · 使用设备**：~~电脑+手机双端~~ → **仅手机**（2026-07-07 策略转向）
- **2 · 数据存放**：**手机本地存储，无需任何服务器/域名/账号**，零成本、离线可用（原"云端部署"方案随之作废）

## 待定问题（按依赖顺序）

- [ ] **2b. 手机平台（当前卡点）**：安卓（含可装 APK 的老鸿蒙）→ Capacitor 打包 APK 直接安装（推荐）；iPhone（用户无 Mac）→ 同一套代码做成 PWA + 免费静态托管，Safari 添加到主屏幕；纯血鸿蒙 NEXT → 需单独评估
- [ ] **2c. 数据备份方案**：数据只在手机上，丢机=丢菜谱，需设计导出/备份（如导出文件、坚果云 WebDAV 自动备份）
- [ ] 3. 使用者范围：仅本人 / 家人共用（影响是否需要账号）
- [ ] 4. 菜谱数据模型：一条菜谱包含哪些字段（配料、步骤、图片、标签、来源、心得……）
- [ ] 5. 核心功能清单与优先级：搜索、标签、拍照、份量换算、购物清单、从网址导入……
- [ ] 6. "精美"的具体方向：视觉风格、参考的 app、深色模式、中文排版
- [ ] 7. 技术栈选型（依赖 2 的答案）
- [ ] 8. 启动与日常使用方式（怎么打开它）
