# 拾味 · 领域词汇（Context）

拾味是一本完全离线的私人菜谱手册（iPhone PWA），数据只存在本机。
本文件是后续会话共享的通用语言表：只记录本项目特有的概念，不含实现细节。

## 菜谱与配料

**菜谱 (Recipe)**：
一道菜的完整记录（名称、封面、配料、步骤、小贴士、烹饪计数等）。数据只存本机。
_Avoid_: 食谱、recipe card

**配料 (Ingredient line)**：
菜谱里的一行「食材 + 用量」，如 `{ name:'食用油', amount:'2汤匙' }`。隶属于某一道菜谱。
_Avoid_: 材料、ingredient（泛指时）

**食材 (Grocery good)**：
可购买的实物本身，由配料的 `name` 归一后标识（如「食用油」）。买菜清单按食材归组。
_注_：归组前先按同义词表归一（`油 / 色拉油 / 植物油 → 食用油`、`西红柿 → 番茄`，可扩充）；表外食材按精确名（trim/去空格）分组。**刻意只并入明确列出的别名**，因此 `香油 / 辣椒油 / 蚝油` 不会被误并进 `油`。
_Avoid_: 材料

**用量 (Amount)**：
配料里描述数量的自由文本，如 `2汤匙`、`半茶匙`、`适量`、`2个（约400g）`。非结构化。
_Avoid_: 数量、分量

**可解析用量 (Parseable amount)**：
整串恰好匹配「数字（含半/小数）+ 已知单位」的用量（如 `2汤匙`、`半茶匙`）。
只有可解析用量之间、且单位相同才相加；其余为**不可解析用量**，按来源罗列。

**单位 (Unit)**：
被识别的烹饪计量词（茶匙 / 汤匙 / 克 / g / 毫升 / ml / 个 / 根 / 片 / 把 / 滴 / 罐 / 斤 / 两 …）。不同单位不互相换算。

## 买菜清单

**买菜清单 / 当前清单 (Shopping List / Current List)**：
由若干选中菜谱聚合出的、可勾选的采购清单。全库同时只有**一份**当前清单，持久化在 meta 库。
_Avoid_: 购物车、采购单

**清单项 (List item)**：
清单里按食材归组的一行。要么显示一个相加后的用量，要么显示多条来源明细。

**来源明细 (Provenance entry)**：
清单项里一条 `{ 用量, 菜名 }`，用于不可解析或跨单位、无法相加时如实展示。

**勾选态 (Checked state)**：
当前清单里哪些食材已买到。按食材名记录，随当前清单持久化。
_注_：与详情页「烹饪模式」里按「菜谱 → 配料下标」记录的 `state.checked` 是**不同**概念，勿混用。

**选菜器 (Recipe picker)**：
`#/shop` 页顶部用于勾选「今天要做的几道菜」的菜谱列表。
