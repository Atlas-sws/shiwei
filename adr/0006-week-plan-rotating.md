# 周菜单为固定周一~周日循环表，不绑日期、不留历史

## Context

周菜单计划（Mealie / Tandoor / KitchenOwl 的标配能力）要选数据形态：绑定具体日期的日历式（可翻周、有历史），还是一张固定「周一~周日」的循环表。日历式功能更全，但实现和 UI 重一档，且历史数据会无限累积——对单人本地 App 的「家里这周吃啥」场景价值存疑。

## Decision

采用**固定七槽循环表**：meta 库 `weekPlan = { days: [七个菜谱id数组] }`，不绑日期、零历史。与买菜清单的联动为**只增不减的并入**：整周或单日把菜谱 id 合并去重进当前清单已选，保留勾选态与手动项，绝不替换或清空用户已有的清单内容。

## Considered Options

- **日历式绑定日期**——被拒：重一档、历史无限累积；「回看上周吃了啥」可由做菜计数/最近做过近似满足。

## Consequences

- 无历史可回看；若将来要历史，需换数据形态（届时另立 ADR）。
- 并入是单向的：从周菜单到买菜清单；买菜清单的变化不回写周菜单。
