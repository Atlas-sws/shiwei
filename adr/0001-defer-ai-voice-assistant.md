# 维持离线·零成本根基，搁置需后端的 AI 语音助手

## Context

有人提出「炒新菜时的实时对话式 AI 语音助手」。真正的对话式 AI 需要后端 LLM、联网、按 token 计费，并需管理 API key；语音识别在 iOS Safari 上还会把音频送到 Apple 服务器。这与拾味的根基决策（完全离线 · 无服务器 · 无账号 · 零成本 · 零依赖，见 [DESIGN.md](../DESIGN.md)）直接冲突。

## Decision

本轮不做 AI 语音助手，维持离线 / 零成本 / 无服务器根基不变。若日后要做，需作为一次**显式的架构决策**单独启动——BYO API key 还是自建代理、联网降级体验、token 成本归属，都要重新决定，而不是顺手加进当前 PWA。

## Consequences

- 「实时对话辅助」被拆解：纯端上的语音体验（TTS 朗读步骤、按钮/语音切换下一步）不需要后端，可作为一条独立的低风险路线单独评估；真正的 AI 对话才触发上面的架构决策。
